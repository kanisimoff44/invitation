# Идентификация приглашённого
Поле с вводом имени и фамилии в форме подтверждения присутствия (RSVP).

# Учёт гостей (вместо интеграции с готовой Google/Яндекс формой)
Вместо встраивания готовой формы — своя форма на странице приглашения, которая отправляется на backend (FastAPI):
- Ответ сохраняется в базу данных на сервере (основной источник данных и бэкап).
- Дополнительно ответ автоматически дублируется в Google-таблицу через Google Sheets API (Service Account), чтобы можно было видеть список гостей в привычном табличном виде без необходимости заходить на сервер.
- Если Google Sheets временно недоступен (сеть/настройка) — ответ гостя всё равно сохраняется в базе, ничего не теряется.

# Счётчик "+1/+2"
В форму подтверждения добавляется поле количества гостей (идёт один / +1 / +2). 
Эта фича реализуется так, чтобы её можно было включить или выключить одной настройкой на сервере (без переписывания кода) — на случай, если после согласования с заказчиком решение изменится.

# Референс стиля
В качестве референса использован https://weddingpost.ru/cabinet/style/variant/?style=262#1163 (страница находится в личном кабинете, доступ ограничен — стиль и анимации продуманы самостоятельно в похожей эстетике: лёгкие появления секций при скролле, декоративные элементы в красно-белой палитре).

# Адаптивность
Страница адаптивная (mobile-first), корректно открывается и в браузере на компьютере, и в браузере на телефоне.

# Технические решения и размещение
- Backend: FastAPI (Python), хранение ответов гостей — SQLite.
- Frontend: серверный рендеринг (Jinja2-шаблоны) + обычные CSS/JS, без отдельного SPA-фреймворка — страница по сути статична, это не требуется.
- Размещение: на уже имеющемся VPS с уже купленным доменом, через Docker Compose (приложение + nginx + certbot).
- HTTPS: сертификат Let's Encrypt, выпускается и автоматически продлевается через certbot.
